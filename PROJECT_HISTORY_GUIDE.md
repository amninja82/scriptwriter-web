# 项目历史管理功能使用指南

## 功能概述

编剧智能体现在支持**项目对话历史的完整保存和跨项目共享**，让您可以：

✅ **切换项目时自动恢复历史对话** - 智能体记住之前所有的创作内容
✅ **跨项目搜索** - 在当前项目中搜索其他项目的对话内容，实现信息共享
✅ **项目历史摘要** - 快速查看项目的关键信息和最近的对话

## 对话历史存储机制

### 存储方式
- 每个项目的对话历史存储在独立的**知识库数据集**中
- 数据集命名：`project_history_{project_id}`
- 对话按时间顺序存储，支持全文检索

### 存储内容
每次对话包含：
- **角色**（user/assistant）
- **内容**（对话文本）
- **时间戳**（ISO 8601格式）

### 存储时机
- 对话历史可以手动保存（使用 `save_conversation_to_project`）
- 未来版本将支持自动保存

## 常见使用场景

### 场景1：多项目并行创作

**流程**：
1. 创建项目A（电视剧），进行创意引导
2. 切换到项目B（电影），开始创作
3. 需要回到项目A时：`切换到项目A`
4. 智能体自动恢复项目A的所有历史对话
5. 继续之前的讨论和创作

### 场景2：跨项目参考

**流程**：
1. 在项目A中创作某个角色设定
2. 在项目B中想参考类似的角色设定
3. 使用 `搜索所有项目中的角色设定`
4. 智能体返回所有相关的内容片段
5. 在项目B中参考这些内容

### 场景3：断点续作

**流程**：
1. 之前创作了项目C，进行了多轮对话
2. 今天想继续创作项目C
3. 使用 `切换到项目C` 或 `进入项目C`
4. 智能体自动恢复所有历史对话
5. 可以无缝继续之前的创作

```markdown

## user (2026-04-24T10:00:00)

## assistant (2026-04-24T10:00:05)
智能体的回复内容

...
```

### 搜索方式

- 支持中文关键词搜索

## 注意事项

1. **对话历史不会自动保存**
   - 当前版本需要手动调用 `save_conversation_to_project` 保存
   - 未来版本将支持自动保存

2. **知识库存储限制**
   - 单次对话建议不超过 2000 字符
   - 大量对话会被分块存储

3. **切换项目的影响**
   - 切换项目后，当前对话上下文会改变
   - 建议在切换前完成当前任务的讨论

4. **跨项目搜索**
   - 搜索结果按相关性排序
   - 可以指定搜索特定项目

## 常见问题

### Q1: 切换项目后，智能体会记得之前的内容吗？
**A**: 是的！切换项目时，智能体会自动加载该项目的所有历史对话，恢复到之前的状态。

### Q2: 可以在不同项目间共享信息吗？
**A**: 可以！使用 `search_all_projects` 命令可以在所有项目中搜索内容，实现跨项目信息共享。

### Q3: 对话历史会占用多少存储空间？
**A**: 每条对话通常占用几百字节到几KB，取决于对话长度。知识库有充足的存储空间。

### Q4: 如何删除项目的历史对话？
**A**: 目前需要使用 `delete_knowledge_document` 工具，指定项目历史数据集和文档ID。

### Q5: 可以来回切换项目吗？
**A**: 完全可以！您可以随时切换到任何项目，智能体都会自动加载对应的历史对话。
