# 多智能体编剧系统 - 网页版使用指南

## 🎉 功能特点

✅ **完全的网页界面** - 无需安装任何软件
✅ **左侧对话历史** - 类似豆包的界面设计
✅ **项目管理** - 可以创建、切换、查看多个剧本项目
✅ **本地存储** - 对话历史保存在浏览器本地
✅ **响应式设计** - 支持电脑和手机访问
✅ **一键使用** - 输入网址即可使用

---

## 📱 使用方式

### 方式1：本地使用（最简单）

#### 步骤1：下载文件

从 Coze Coding 下载 `scriptwriter_web.html` 文件

#### 步骤2：双击打开

```
直接双击 scriptwriter_web.html 文件
浏览器会自动打开
```

#### 步骤3：输入 Token

```
在左下角的输入框中粘贴您的 API Token
点击页面任意地方确认
```

#### 步骤4：开始使用

```
✅ 可以创建新剧本项目
✅ 可以查看已有项目
✅ 左侧显示对话历史
✅ 完全在浏览器中使用
```

---

### 方式2：GitHub Pages（免费网址）

#### 步骤1：准备文件

1. 创建一个 GitHub 账号（如果没有）
2. 创建一个新仓库
3. 上传 `scriptwriter_web.html` 文件

#### 步骤2：启用 GitHub Pages

```
1. 进入仓库设置
2. 找到 "Pages" 选项
3. 选择 "Source: Deploy from a branch"
4. 选择分支（通常是 main 或 master）
5. 点击 "Save"
```

#### 步骤3：获取网址

```
等待1-2分钟后，GitHub 会生成一个网址：
https://您的用户名.github.io/仓库名/

例如：https://zhangsan.github.io/scriptwriter/
```

#### 步骤4：使用网址

```
1. 复制这个网址
2. 在浏览器中打开
3. 输入 Token
4. 开始使用
```

---

### 方式3：Vercel/Netlify（更专业的托管）

#### 使用 Vercel

```
1. 访问 https://vercel.com
2. 登录（可以用 GitHub 账号）
3. 点击 "New Project"
4. 导入 GitHub 仓库
5. 自动部署
6. 获得一个 .vercel.app 域名
```

#### 使用 Netlify

```
1. 访问 https://netlify.com
2. 登录（可以用 GitHub 账号）
3. 点击 "Add new site" → "Deploy manually"
4. 拖入 scriptwriter_web.html 文件
5. 自动部署
6. 获得一个 .netlify.app 域名
```

## 💡 功能使用

### 1. 创建新剧本项目

```
方法1：点击左侧 "+ 新建剧本项目"
方法2：在欢迎页面点击 "🎬 创建新剧本项目"
方法3：直接输入消息开始对话
```

### 2. 查看已有项目

```
在左侧"最近对话"列表中点击项目
可以继续之前的创作
```

### 3. 切换项目

```
点击左侧列表中的不同项目
右侧会显示对应的对话历史
```

### 4. 输入 Token

```
在左下角的输入框中粘贴 API Token
Token 会保存在浏览器本地
下次打开无需重新输入
```

## 🔐 安全说明

### Token 安全

```
✅ Token 保存在浏览器 localStorage
✅ 不会上传到其他服务器
✅ 只在您的设备上使用
⚠️ 不要在公共电脑上使用
⚠️ 不要分享您的网址（如果包含 Token）
```

### 数据隐私

```
✅ 对话历史保存在浏览器本地
✅ 不会上传到其他服务器
✅ 可以随时清除浏览器数据
✅ 不同设备之间不会同步
```

## 📊 数据存储

### 本地存储位置

```
浏览器 → 开发者工具 → Application → Local Storage

存储内容：
- scriptwriter_token: API Token
- scriptwriter_chats: 对话历史
```

### 清除数据

```
方法1：浏览器清除数据
  → 设置 → 隐私 → 清除浏览数据

方法2：开发者工具
  → Application → Local Storage → 右键清除

方法3：在页面中（需要添加清除按钮）
  → （可以联系我们添加这个功能）
```

## ❓ 常见问题

### Q1: 网址可以分享给他人吗？

```
⚠️ 可以分享，但需要注意：
  - Token 会保存在各自浏览器中
  - 对话历史不会同步
  - 建议每个人都输入自己的 Token
```

### Q2: 对话历史会丢失吗？

```
✅ 不会，保存在浏览器本地
❌ 清除浏览器数据会丢失
❌ 换设备需要重新开始
```

### Q3: 可以在手机上使用吗？

```
✅ 可以！响应式设计，支持手机访问
✅ 左侧边栏可以折叠（移动端）
✅ 触摸屏优化
```

### Q4: 可以导出对话历史吗？

```
当前版本：暂不支持
可以添加此功能（联系我们）
```

### Q5: 网址安全吗？

```
✅ HTTPS 加密
✅ 不会收集数据
✅ Token 保存在本地
✅ 只调用我们的 API
```
